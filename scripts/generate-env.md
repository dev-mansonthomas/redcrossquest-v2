# generate-env.sh

## Description

Génère les fichiers `.env` des différents composants du projet (backend, superset, provisioning, frontend) à partir du fichier de configuration racine (`.env` ou `.env.<env>`). Ce script centralise la configuration et distribue les variables appropriées à chaque composant.

## Prérequis

- **Bash** (version 4+)
- Un fichier de configuration source à la racine du projet :
  - `.env` pour l'environnement `local`
  - `.env.dev` pour l'environnement `dev`
  - `.env.test` pour l'environnement `test`

## Usage

```bash
./scripts/generate-env.sh <environnement>
```

### Environnements supportés

| Environnement | Fichier source | Description |
|---------------|---------------|-------------|
| `local`       | `.env`        | Développement local (Docker) |
| `dev`         | `.env.dev`    | Environnement de développement GCP |
| `test`        | `.env.test`   | Environnement de test GCP |

> ⚠️ L'environnement `prod` n'est **pas** supporté par ce script.

## Exemples d'utilisation

```bash
# Générer les fichiers .env pour le développement local
./scripts/generate-env.sh local

# Générer les fichiers .env pour l'environnement dev
./scripts/generate-env.sh dev
```

## Fichiers générés

| Fichier | Environnements | Description |
|---------|---------------|-------------|
| `backend/.env` | tous | Configuration du backend FastAPI |
| `superset/.env` | tous | Configuration de Superset (Docker Compose) |
| `superset/provisioning/.env.<env>` | tous | Configuration du provisioning Superset |
| `frontend/src/environments/environment.local.ts` | `local` uniquement | Configuration Angular (TypeScript) |

### Comportement spécifique par environnement

- **local** : Les URLs Superset et frontend utilisent des adresses Docker internes ou localhost. L'URI SQLAlchemy pointe vers le conteneur `rcq_mysql`. Le fichier `environment.local.ts` est également généré.
- **dev / test** : Les URLs proviennent des variables d'environnement du fichier source.

## Notes

- Tous les fichiers générés portent l'en-tête `# AUTO-GENERATED by generate-env.sh — DO NOT EDIT MANUALLY`.
- Le script utilise `set -euo pipefail` : il s'arrête à la première erreur.
- Le fichier `environment.local.ts` n'est généré que pour l'environnement `local`.
- Ce script est appelé automatiquement par `run_local.sh` et `gcp-deploy.sh`.
